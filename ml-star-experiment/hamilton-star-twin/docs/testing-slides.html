THE BIG IDEA

Three layers.

01

SCXML + physics unit tests.

Per-command fidelity against VENUS source line numbers. C0AS, C0DS, C0TP, C0TR, motion envelope, iSWAP plate dims, tip-mask isolation. Cheap, fast, every PR.

02

Bridge integration tests.

Same TCP path VENUS uses — plain BDZ line-delimited. Asserts end-state deltas (well volumes, tip counts, motion envelopes). CI-gated.

03

VENUS ComTrace replay.

Byte-level regression against .trc recordings. Shape-matches each id#### response's prefix + error shape + named-field set. Local only (corpus lives next to the working copy, not in git).

Why shape, not value

What shape check looks at

  • prefix — the 4-char C0XX code
  • id#### — 4-digit order id must match
  • er## / er##/## — error shape equivalence
  • fields[] — set of 2-char named keys must be a superset

What it ignores

  • Numeric values (grip force, Z position, serial)
  • Field order (keys compared as sets)
  • Whitespace + end-of-line
RATIONALE
Real hardware returns 0x0ACB grip force, 1780 Z position, serial SN-1234567 — the twin can't reproduce those bit-for-bit. But if the twin drops the lh field, VENUS's parser returns undefined for "last liquid level" and the next command silently rejects. Shape-match catches that.

Parsing a response down to its fingerprint

interface ParsedResponse {
  prefix: string;               // module+code, e.g. "C0RF"
  orderId: string | null;       // "0004"
  error: { main: string; detail: string | null } | null;  // "er00" or "er25/02"
  fields: string[];             // ordered list of named 2-char keys
}

// Field extraction: 2-char lowercase token followed by a non-alpha char.
// Filters out incidental letter pairs that are part of a value.
const fieldRe = /([a-z]{2})(?=[^a-z]|$)/g;

// Success = actual shape is a SUPERSET of expected. Twin may emit extra
// debug fields, but must never DROP a field VENUS reads.
for (const f of expected.fields) {
  if (!actual.fields.includes(f)) return `missing field ${f}`;
}
First 30 pairs of TipPickup1ml_ComTrace.trc covers every module's init handshake.
Break any response shape → test fails with every divergence reported at once.

Anatomy of a per-command test

it("aspirate tm=2 leaves ch0 volume untouched", () => {
  twin = createTestTwin();                             // 1. sync in-process
  const tip = twin.wellXY("TIP001", 0, 0, 0);
  twin.sendCommand(`C0TPid0001xp${tip.xp}yp${tip.yp}tm255tt04tp2264tz2164th2450td1`);
  twin.fillPlate("SMP001", 0, "water", 15000);         // 2. arrange state
  const smp = twin.wellXY("SMP001", 0, 0, 0);

  // Aspirate 500 on ch0 (tm=1)
  twin.sendCommand(`C0ASid0002xp${smp.xp}yp${smp.yp}av00500tm01lm0`);
  const vol_after_ch0 = twin.getState().modules.pip!.variables.volume as number[];
  expect(vol_after_ch0[0]).toBe(500);                  // 3. exact equality
  expect(vol_after_ch0[1]).toBe(0);                    //    not toBeGreaterThan

  // Aspirate 300 on ch1 (tm=2) — ch0 must STAY at 500
  twin.sendCommand(`C0ASid0003xp${smp.xp}yp${smp.yp}av00300tm02lm0`);
  const vol_after_ch1 = twin.getState().modules.pip!.variables.volume as number[];
  expect(vol_after_ch1[0]).toBe(500);   // ← before fix: gets overwritten with 300
  expect(vol_after_ch1[1]).toBe(300);   // 4. assert the delta, not just "accepted"
});

PRINCIPLE 04

The twin fits in a test

createTestTwin() — one function, full twin

// tests/helpers/in-process.ts
export function createTestTwin(options: CreateTestTwinOptions = {}): TestTwin {
  const { autoInit = true, ...deviceConfig } = options;
  const api = new DigitalTwinAPI();            // runtime loaded from dist/ (CJS)
  const deviceId = api.createDevice({ name: "TestTwin", ...deviceConfig });

  return {
    deviceId, api,
    sendCommand:    (raw)  => api.sendCommand(deviceId, raw),
    sendCompletion: (evt)  => api.sendCompletion(deviceId, evt),
    flushPending:   ()     => api.flushPendingEvents(deviceId),
    reset:          ()     => api.resetDevice(deviceId),
    initAll,          // auto-runs C0VI/C0DI/C0EI/C0II on construction
    getState, getModuleStates, getModuleVars,
    getTracking, getAssessments,
    wellXY,           // resolves "SMP001", pos 0, col 3, row A → { xp, yp }
    fillPlate,        // arrange liquid-tracker state before a test
    onEvent, destroy,
  };
}
WHY DIST/, NOT SRC/
The SCXML modules are generated as ESM import + CJS require hybrids. The runtime loads them via require(), which needs the dist/ build output (plus scripts/convert-modules.js). Testing against dist/ also means we test what Electron / headless actually ships.

Helpers that keep tests readable

twin.sendCommand(`C0TPid0001xp${tip.xp}yp${tip.yp}tm255tt04`);

PRINCIPLE 05

Failure injection or it doesn't count

Every new test proves it can catch a regression

FAILURE INJECTION

Prose at the top of each test file listing the bugs this test would catch. Reviewer's checklist: "If I break X, does this fail?"

EXAMPLE

"If computeStateAt skips the re-execution step, jump(N).state equals initialState for all N — the aspirate-changes-volume case fails."

/**
 * Per-channel state isolation for C0TP / C0AS / C0DS / C0TR.
 *
 * FAILURE INJECTION
 *   - If C0AS overwrites volume[i] for EVERY fitted channel (not just
 *     masked), "aspirate tm=2 leaves ch0 volume untouched" fails.
 *   - If C0TR ejects ALL fitted channels on a partial mask, "partial
 *     eject (tm=1) keeps other channels' tips" fails.
 *   - If C0TP rejects any second pickup (error 07) instead of accepting
 *     non-overlapping masks, "pickup subset then add more" fails.
 */

Banned patterns

PatternWhy it's bannedUse instead
expect(result.accepted).toBe(true) Command was accepted — says nothing about physics. Capture before + after state; assert the delta.
expect(vol).toBeGreaterThan(0) Passes for wrong values (1, 500, 5 000 all "greater than 0"). toBe(500) — exact equality.
expect(state).toBeDefined() Always true when no exception thrown. Assert specific fields + values.
Mocking sendCommand Tests the mock, not the twin. Use createTestTwin().
RULE
A test that never fails against bugs is worse than no test. The tests/TESTING-GUIDE.md discipline is reviewable: every PR adds tests with an explicit "what would break this" paragraph.

WHY THE TWIN WANTS TO BE "REAL"
End-to-end tests driving VENUS 6.0.2 against Simulation would exercise VENUS's stub path — no FW pipeline, no physics, no rejections. Silent success proves nothing. The twin earns "real" treatment by speaking the BDZ discovery handshake byte-for-byte against VENUS's parser.

Unlocks full pipeline

  • Discovery: UDP unicast reply with exact bdc_* field list
  • Enumeration: TCP reply with r_device_interfaceprotocol 2
  • FW: init sequence responds with VENUS-expected fields (rf, kb, lh, ru, …)

Drops to Simulation

  • Missing / misspelled bdc_modulename field
  • r_device_interfaceprotocol ≠ 2 (only FDx accepted by the STAR config)
  • Init command drops a named field → VENUS can't parse → device rejected

Testing implication

Bridge tests

tests/unit/discovery-server.test.ts + fdx-init-responses.test.ts pin every byte VENUS's parser reads. Break one of these, VENUS drops the twin from the picker — silently.

Method-level E2E

runVenusMethod(..., via: "venus-web-api") only makes sense against a twin VENUS sees as real. In Simulation mode, there's nothing to test — VENUS isn't calling FW.

CATCHING THE SILENT DROP
The worst failure mode is VENUS quietly picking Simulation instead of the twin — tests pass internally but the "real VENUS" end-to-end ran against a stub. The .trc replay harness + VENUS-6.0.2-sourced init responses are the fence: the twin can't shape-match a real recording and still be simulation-treated.

WHY BOTH
VENUS traces prove the twin matches real firmware. Twin traces prove the PSI analysis stack (phase-3 replay, what-if, scrubber) reconstructs state losslessly. Same principle — recordings as regression oracles — applied at two different layers.

The replay invariant

// tests/unit/trace-replay-service.test.ts
it("jump(N) followed by step(forward) equals jump(N+1)", () => {
  const trace = buildTrace();             // record fresh
  const replayA = new TraceReplayService(trace);
  const replayB = new TraceReplayService(trace);

  for (let n = 1; n < trace.events.length - 1; n++) {
    replayA.jump(n);
    replayA.step("forward");
    replayB.jump(n + 1);

    expect(replayA.state).toEqual(replayB.state);   // ← the property test
  }
});
Breaks if computeStateAt skips re-execution, if snapshots miss a field, or if the event spine drops correlation IDs. One test, three classes of regression.
